# A Comparative Study of Moral Framing in Real and LLM-Generated Partisan Climate Headlines

This repository contains the data, code, and paper for a Cognitive Data Science project at the
University of Trento. The study looks at how US climate news headlines from Democrat- and
Republican-leaning audiences differ in their *moral framing*, and whether headlines written by
a large language model reproduce that structure.

The short version of the result: at the level of moral foundations the two real audiences are
almost indistinguishable — what tells them apart is subject matter, not morality — while the
LLM-generated corpus is clearly more moralised and more polarised than the real headlines it
was asked to imitate.

## Data

- **`earth_day_real_articles-2.json`** — real climate headlines collected from the Media Cloud
  API for 22 April 2016 and 2017. Articles are grouped by partisan-audience quintile (Q1–Q5),
  where an outlet's quintile reflects the partisanship of the audience that shares it. Q1–Q2 are
  treated as the Democrat-leaning group and Q4–Q5 as the Republican-leaning group. The
  collection code is in `Hackathon_Notebook_22_APRIL.ipynb`.
- **`dem_titles_final.json` / `rep_titles_final.json`** — synthetic headlines generated by
  Claude Sonnet 4.6, matched to the real counts (Democrat 363, Republican 113). The prompt is in
  `prompts/generation_prompt.txt`.

Two dictionaries used by the analysis are **not** stored here, because they are third-party
resources with their own licences:

- **eMFD** — extended Moral Foundations Dictionary (Hopp et al., 2021), MIT-licensed.
- **LibertyMFD** — liberty-foundation lexicon (Araque et al., 2022), LGPL-3.0.

`analysis.py` downloads both automatically the first time it runs, so nothing else needs to be
done by hand.

## How to run it

```bash
python -m venv .venv
source .venv/bin/activate          # on Windows: .venv\Scripts\activate
pip install -r requirements.txt

python analysis.py                 # prints every number in the paper and writes CSVs to outputs/
python figures.py                  # writes Figure 1 and Figure 2 to outputs/
```

The required NLTK data (tokeniser, tagger, lemmatiser, stopwords) is downloaded on the first
run. Everything is seeded, so the numbers reproduce exactly.

## What the scripts produce

`analysis.py` writes, to `outputs/`:

- `table1_moral_means.csv` — mean eMFD foundation probabilities for the four groups, with the
  individualising/binding composites (Table 1 in the paper).
- `table2_fidelity.csv` — the fidelity diagnostics: partisan-gap significance in each corpus and
  the real-vs-synthetic effect sizes (Table 2).
- `real_partisan_effectsizes.csv` — Cohen's *d* and 95% bootstrap confidence intervals for the
  real Democrat-vs-Republican contrast.
- `liberty_robustness.csv` — the LibertyMFD result across all three released lexicon versions.
- `logodds_real.csv` — the weighted log-odds vocabulary underlying Figure 2.

`figures.py` writes `figure1_real_vs_synth.png` (moral-foundation profiles) and
`figure2_real_frame.png` (the real partisan vocabulary).

## Method, briefly

Titles are lemmatised and reduced to content words. Moral content is scored with the eMFD for the
five classic foundations and with LibertyMFD for the sixth (liberty). Democrat–Republican and
real–synthetic contrasts are tested with two-sided permutation tests (20,000 resamples), reported
with Cohen's *d*, 95% bootstrap confidence intervals, and Bonferroni correction across the five
foundations. Weighted log-odds with an informative Dirichlet prior (Monroe et al., 2008) recover
the vocabulary that distinguishes the two audiences.

## Reproducing the synthetic corpus

The synthetic titles were generated once, with **Claude Sonnet 4.6**, using the prompt in
`prompts/generation_prompt.txt` (one run per party). The prompt fixes the date, the count, the
climate keywords, the partisan alignment, and the instruction to invent rather than retrieve.
